1. um ponteiro é uma variável que aponta para um endereço de memória de outra variável. endereço de memória descreve em qual logal da mem´ria que uma determinada variável está. ponteiros são importantes porque permitem a modificação no valor de variáveis indiretamente e/ou somente numa determinada parte do código.
2.  ponteiro nulo é aquele que não aponta para nada e é utilizado no incio de um programa para evitar que o ponteiro que sera utilizado somente posteriormente não aponte para nehum lixo de memória. da erros pq ele não aponta pra nada
3.  * na declaração: indica que é um ponteiro, ou seja, que vai armazenar um endereço de memória. * na desreferenciação: faz com que acesse o valor que o ponteiro esta apontando. ex: correto:  int num = 23 ; int *ptInt = &num; printf("%p", ptInt) ; incorreto: int *ptInt = 23; printf("%p", *ptInt); 
4.  a modulirazação ajuda a resolver problemas maiores divididndo eles em partes menores, isso faz com que o problema seja resolvido com mais eficiencia e sem deixar lacunas. clareza: mais fácil de etender. manutenção: mais fácil de modificar qualquer estrutura. reutilização: uma função pode ser chamada (utilizada) várias vezes alo longo de um programa. 
5.  tipo nome (parametros - passados por valor ou referência) e {comandos}. se for tipo void, nao precisa retornar nada, se for qualquer outro tipo, precisa retornar um valor. return tem o papel de retornar um valor (ou nao) quando a função para.
6.  formais: definidos na função. reais: usados na função main na chmada da função desejada. o casamento ocorre quando no ato dessa chmada na main: os valores que são passados na main subtituem aqueles dentro da função. ex.: void minhaFuncao (num1, num2) {...}
int main () {....minhaFuncao(n1, n2);...}. ex incorreto: void minhaFuncao (num1, num2) {...}
int main () {...minhaFuncao(n1, n2, n3);...}
7.  passagem por valor é quando a funcao cria uma copia daquela varivel e faz as operações com essa cópia. nao alteram pq nao mexe com valor daquela variavel.
8.  atraves dos ponteiros, podemos modificar o valor dentro de determinada variaveis indiretamente pq a funcao recebe o endereço da variavel origirinal.
9.  caso base: é a condição de parada desse tipo de função. caso rescursivo: é quando o valor nao esta dentro da condicao de parada e continua sendo modificado dentro da chmada da função dentro da propria função.
10.  da erro na execução: pode gerar uma recurssão infinita, desencadeando estouro de pilha. estouro de pilha é quando uma funcao é chamda tantas vezes que estoura o limite da pilha reservada para o programa.
11.  entrada: parametro so leva o valor pra dentro da função mas não o modifica. saída: parametro so serve pra gerar um resultado dentro da função, ela nao usa o valor pasado, so escreve nele atraves de um ponteiro. entrada/saida: funcao recebe o valor como parametro e o modifica.
